**Compréhension Globale**
Le projet est un monorepo Laravel séparé en deux applications :

* `api/` : backend REST Laravel 11, responsable des données, auth, catalogue, panier, commandes, TVA, SEO, admin.
* `web/` : frontend Laravel 11 + Livewire + Blade + Tailwind + Alpine, responsable de l’expérience client.
* La communication se fait via HTTP REST : le web appelle l’API avec `ShopApiClient` et `AccountApiClient`.

Le cahier de charge `text.txt` décrit une plateforme e-commerce Europe ambitieuse : catalogue, panier, checkout, paiement Stripe/PayPal, TVA multi-pays, RGPD, livraison, SEO, compte client, admin, performance, tracking, etc. Point important : plusieurs cases sont marquées `✅` dans le cahier, mais dans le code actuel, elles ne sont pas toutes réellement terminées. Certaines sont posées comme architecture ou MVP prévu.

**Lecture Parano Du Cahier De Charge**
Le cahier est bon comme vision, mais il mélange trois niveaux :

* Ce qui est déjà codé.
* Ce qui est prévu dans l’architecture.
* Ce qui est souhaité comme produit final.

Les points les plus sensibles :

* Le cahier parle de paiement Stripe/PayPal comme fait, mais le code ne montre pas encore de paiement réel.
* Le cahier parle de transporteurs DPD/GLS/UPS, alors que votre vrai besoin récent est Mondial Relay + Chrono Relais. Il faut corriger officiellement le cahier.
* Le checkout demandé est `adresse -> paiement -> confirmation`, mais votre besoin réel doit être `panier -> adresse -> transporteur -> point relais -> devis -> paiement -> confirmation`.
* La TVA est mieux avancée que le paiement, mais elle doit être validée juridiquement pour les pays UE.
* Le web affiche un bon tunnel UX, mais il doit maintenant être branché aux vrais endpoints `checkout/quote` et `orders`.
* Les produits seedés sont encore génériques, pas encore les vrais produits DEN & FILS / Selakay.
* L’architecture est saine, mais le projet n’est pas encore “prêt production” tant que paiement, livraison réelle, commande web et tests E2E ne sont pas terminés.
